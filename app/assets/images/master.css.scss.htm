Lesson 7: Styling the Top Navigation

Our application has a static splash page that includes Twitter Bootstrap and includes a top navigation. In this lesson, we will adjust the visual display of the top navigation by adjusting our CSS styling.

Classmates working on this track:


If you think back to the Nomster application, making a few font adjustments inside the top navigation made it look a little sharper. I'm going to use Helvetica as our top navigation font for now.

If you're not a Helvetica fan, have a look at the previous Nomster Font Advice and choose another font stack for your web app.

By now you should be fairly confident with CSS and styling. In a nutshell we basically create a CSS class (like .navbar) and add specific styling elements inside it (like border-radius: 0;).

If you want to refresh your memory on how CSS classes and styles work, have a look at lesson 9 of the Nomster class.

Lets get coding

The first thing that will make our top navigation look a little better is to remove the rounded corners of it. We can do that by adjusting the border-radius to 0. Let's do that by creating app/assets/stylesheets/master.css.scss and adding the following code: 

.navbar {
  border-radius:0;
}

Save the file, refresh the page and make sure that the rounded corners are gone. Cool!

Next up, lets switch the font in the navbar to be Helvetica with a weight of 200. Adjust master.css.scss to include the following: 

.navbar {
  border-radius:0;

  font-family: "Helvetica Neue", Helvetica;
  font-weight: 200;
}

Save the file and refresh the page. Nice, it looks already a bit better.

Now I'm going to adjust the font of the brand (the thing in the top left of the navbar that says Flixter for me). I'm going to make mine be a very thin version of the Helvetica font. 

.navbar {
  border-radius:0;
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 200;
}


.navbar-inverse a.navbar-brand {
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 100;
}

Next change I want to do is make the brand be white instead of grey. 

.navbar {
  border-radius:0;
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 200;
}

.navbar-inverse a.navbar-brand {
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 100;

  color:white;
}

Another thing I really liked about the Nomster top navigation was that little box shadow it had on it. Let's add that to our Flixter navbar too. 

.navbar {
  border-radius:0;
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 200;

  box-shadow: 0px 1px 3px #2a2a2a;
}

.navbar-inverse a.navbar-brand {
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 100;
  color:white;
}

Now the last thing I'm going to do to the top nav is add some small design detail of adding a simple a pattern as the background.

I went to subtlepatterns.com and found a pattern called "dark wood". Find your own pattern, unzip the file and copy the image into your app/assets/images folder.

Now that we have our top navigation pattern saved, let's use it: 

.navbar {
  border-radius:0;
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 200;
  box-shadow: 0px 1px 3px #2a2a2a;

  background:url('dark_wood.png');
}

.navbar-inverse a.navbar-brand {
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 100;
  color:white;
}

Save the file and refresh the page. The top navigation now looks awesome!

How to get a fast answer

How to get a fast answer for your question:

Here’s how to ask smart questions in the forum:


First, give it a good title that describes what you’re struggling with. Descriptive headlines like “my quotes don’t show on my index page“ or “the email notification email is not triggered at comment submission” work best.


Second, describe in detail what you’ve done up to this point. For example, you should say things like: “I added a quote through the rails console and can see it in my database. I also added the code to create the loop on my index.html.erb page and added the necessary code inside of my quotes controller too”.


Third, describe in detail what you expected to happen. For example, you should describe things like this: I refreshed my browser window with the url “localhost:3030” and expected a new quote to be shown on my root page.


Fourth, paste a screenshot of your error message. This one is obvious, if we can see what you’re seeing we can help you a lot better. Make sure you get the whole page.


Fifth, include a link to your github repository: If we can’t figure out how to help you through your description above, we often times go directly into GitHub and take a look at your web application code, pull it onto our machines, and give it a little test drive.

Important: Make sure you always commit and push the latest version of your code to your GitHub repository, before submitting your question in the Q&A forum. By doing so we can look at your latest code and see the same error message as you.


Sixth, include a link to you web application: We might need to have a look at your app live on the internet. Including the url for your app will help. Plus if you don't include the url, you might get a response from Jarvis, the firehose robot, requesting that you add it.


Speed of help: We generally check out forum several times each day and answer all questions. The better you formulate your problem, the faster you get your solution.

This makes sense. If you forget to include a screenshot of your error message, the most likely reply that you get back will be something like “hey there, can you please include a screenshot of your error message so we can see what you’re seeing?”.


Directly emailing your Mentor a question: The forum is a lot faster way to get help - since we always have somebody on dedicated forum watch. It might seem tempting to go around the forum and email your mentor directly, but that will most likely get you a delayed response, since your mentor might be helping other students and will not get back to you as fast.
